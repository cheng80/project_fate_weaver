# Project FateWeaver MVP Loop Validation Plan v0.1

## 문서 목적

이 문서는 Project FateWeaver의 첫 검증 목표를 정의한다.

목표는 Flutter 앱 제작이 아니라, **이벤트-선택-상태 변화 루프가 재미있는지**를 최소 비용으로 검증하는 것이다.

---

12~15개 이벤트만으로도 선택과 상태 변화가 재미있는지 검증한다.

# 6. 테스트 기준

## 6.1 플레이 테스트 기준

최소 5회 이상 반복 실행한다.

확인 질문:

- 같은 이벤트가 다시 나와도 선택이 달라지는가?
- 현재 상태 때문에 선택을 고민하게 되는가?
- 아이템 보유 여부가 선택을 바꾸는가?
- 저주를 낮추는 것이 항상 정답인가?
- “한 이벤트만 더”라는 감각이 있는가?
- 죽었을 때 원인이 납득되는가?

---

## 6.2 통과 기준

다음 중 4개 이상 만족하면 루프 가능성 있음.

- 플레이어가 5회 이상 반복 테스트할 이유를 느낌
- 선택 후 후회가 발생함
- 아이템이 실제 판단을 바꿈
- 저주가 런 흐름을 바꿈
- 이벤트가 단순 수치 손익표로만 느껴지지 않음
- 정답 고정 이벤트가 절반 이하
- 다음 이벤트에 대한 기대가 생김

---

## 6.3 실패 기준

아래 중 하나라도 강하게 발생하면 Constitution 수정 필요.

- 항상 같은 선택이 정답
- 아이템이 없으면 선택지가 단조로움
- 아이템이 있어도 사실상 정답만 늘어남
- 저주가 단순 패널티
- 이벤트 반복 시 암기 게임이 됨
- 전투형 이벤트가 수치 손익표처럼 느껴짐
- 상태 변화가 다음 선택에 거의 영향 없음

# 7. 검증 후 결정

MVP-0 이후 결정한다.

| 루프 재미 있음 | Event Grammar + YAML Schema 작성 |

| 아이템 약함 | Item Role Taxonomy 수정 |
| 저주 약함 | Curse System 축소 또는 재설계 |
| 이벤트 반복 피로 큼 | Event Variation Rule 추가 |
| 전투형 이벤트 약함 | Combat Policy 재검토 |
