# Project FateWeaver MVP Loop Validation Plan v0.2

## 문서 목적

이 문서는 Project FateWeaver의 MVP-0 검증을 위한 실험 프로토콜이다.

목표는 Flutter 앱 제작이 아니라, **이벤트-선택-상태 변화 루프가 재미있는지 측정 가능한 방식으로 검증하는 것**이다.

---

# 1. 검증 단계 정의

## MVP-0: Console Loop Validation

목표:

```text
12~15개 이벤트만으로도 선택과 상태 변화가 재미있는지 검증한다.
```

포함:

- 콘솔 또는 간단한 텍스트 UI
- 임시 하드코딩 또는 Draft YAML 데이터
- 상태 5개
- 아이템 5~6개
- 이벤트 12~15개
- 선택지 3개 기본 구조
- 결과 적용
- 다음 이벤트 선택
- 로그 저장

제외:

- Flutter UI
- Flame 연출
- 카드 애니메이션
- 저장/로드
- 광고/상점
- 별도 전투 시스템

# 7. 테스트 프로토콜

## 7.1 테스트 방식

1차는 Solo Heuristic Test로 진행한다.

| 항목 | 기준 |
|---|---|
| 테스터 | 개발자 1명 |
| 최소 실행 회차 | 5회 |
| 권장 실행 회차 | 10회 |
| 1회 목표 길이 | 10~20 이벤트 |
| 로그 저장 | 필수 |
| 회고 작성 | 각 Run 종료 후 필수 |

2차는 가능하면 외부 2명에게 같은 테스트를 진행한다.

---

## 7.2 테스트 절차

각 Run은 아래 절차를 따른다.

```text
1. 초기 상태와 아이템 확인
2. 이벤트 표시
3. 선택지 확인
4. 선택 전 고민 기록
5. 선택 실행
6. 결과 확인
7. 상태 변화 기록
8. 다음 이벤트 진행
9. Run 종료 후 회고
```


# 9. Run 종료 회고 양식


# 10. 측정 지표

## 10.1 정량 지표

| ID | 지표 | 계산 방법 | 통과 기준 |
|---|---|---|---|
| M01 | 재시작 의향 | Run 종료 restart_intent 평균 | 4/5 이상 |
| M02 | 실패 납득도 | fairness_score 평균 | 4/5 이상 |
| M03 | 정답 고정 이벤트 비율 | fixed_answer_events / 전체 이벤트 | 50% 이하 |
| M04 | 아이템 영향 비율 | 아이템 때문에 선택이 바뀐 이벤트 / 전체 이벤트 | 30% 이상 |
| M05 | 상태 영향 비율 | 상태 때문에 선택이 바뀐 이벤트 / 전체 이벤트 | 30% 이상 |
| M06 | 저주 전략 변화 횟수 | 저주 때문에 선택을 바꾼 횟수 | 3회 이상 |
| M07 | 선택 후회 횟수 | regret_score 4 이상인 선택 수 | 3회 이상 |
| M08 | 미래 영향 비율 | 다음 이벤트/상태에 영향 준 선택 / 전체 선택 | 30% 이상 |
| M09 | 이벤트 피로 | most_boring_event 반복 빈도 | 특정 이벤트 30% 이상이면 수정 |
| M10 | 전투 손익표 느낌 | 전투 이벤트가 단순 수치교환으로 기록된 비율 | 50% 이하 |

## 10.2 정성 지표

테스트 노트에서 아래 문장이 3개 이상 자연스럽게 나와야 한다.

```text
이번엔 다른 아이템으로 시작해보고 싶다.
이번엔 저주를 일부러 쌓아보고 싶다.
이번엔 평판을 버리고 돈을 모아보고 싶다.
이번 선택을 나중에 후회할 것 같다.
다음 이벤트가 궁금하다.
한 이벤트만 더 보고 싶다.
```

# 11. 통과 기준

아래 조건을 모두 만족하면 MVP-0 통과로 본다.

1. 정량 지표 10개 중 7개 이상 통과
2. Critical 실패 기준 미발생
3. 저주 시스템 검증 로그 존재
4. 아이템 영향 비율 30% 이상
5. 정답 고정 이벤트 50% 이하
6. Run 종료 후 재시작 의향 평균 4/5 이상

---

# 12. 실패 기준

아래 중 하나라도 강하게 발생하면 설계 수정이 필요하다.

- 항상 같은 선택이 정답
- 아이템이 없으면 선택지가 단조로움
- 아이템이 있어도 사실상 정답만 늘어남
- 저주가 단순 패널티
- 이벤트 반복 시 암기 게임이 됨
- 전투형 이벤트가 수치 손익표처럼 느껴짐
- 상태 변화가 다음 선택에 거의 영향 없음
- 재시작 의향 평균 3 미만
- 실패 납득도 평균 3 미만

# 13. Failure Review

MVP-0 실패 시 바로 시스템을 확장하지 않는다.

먼저 아래 순서로 원인을 분류한다.

1. 이벤트 문장이 약한가?
2. 선택 구조가 약한가?
3. 상태 영향이 약한가?
4. 아이템 역할이 약한가?
5. 저주가 단순한가?
6. 이벤트 수가 부족한가?
7. 루프 자체가 약한가?

이 순서로 수정한다.
