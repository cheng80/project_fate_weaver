# [Current] Codex P1 First Task Brief v0.1

> 상태: [Current] P0 playable milestone 이후 P1 첫 작업을 Codex가 바로 이어서 수행할 수 있도록 정리한 brief 초안.

## 1. Recommended Task

```text
CODEX_TASK_P1_Human_Playtest_Scenario_Pack_v0.1.md
```

## 2. Why This Task

P0 is automatically playable. P1 should first prove whether a human can read the Text MUD log, understand the goal, make meaningful choices, and report where the loop feels repetitive or weak.

Do not start with broad content expansion or UI. They depend on knowing what the human playtest must measure.

## 3. Task Goal

Create a human playtest scenario pack for the current P0 loop.

Expected outputs:

- 3-5 playtest-ready runs.
- Seed/scenario/profile for each run.
- Expected observations for each run.
- Player feedback questions.
- P0 regression guard.
- Evidence capture instructions.

## 4. Scope

Allowed:

- Documentation.
- Evidence under `.omo/ulw-loop/evidence/`.
- Optional playtest transcript/log references generated by existing commands.

Not allowed:

- Gameplay logic changes.
- Data changes unless the eventual task explicitly scopes a separate P1 scenario pack.
- Test/tool changes.
- Balance/scoring/selection/director/ontology changes.
- Quest/Card/Event/Item/Ending additions.
- Text MUD polish.

## 5. Required Baseline

Before drafting the playtest pack, reconfirm:

- Seed 202 Standard Run remains 25-35 turns.
- Current frozen baseline remains 25 turns unless intentionally rebaselined.
- Result is `success`.
- Ending is `prepared_frontier_route`.
- Three cards appear every turn.
- JSON and Text MUD logs are generated.
- Scenario and ontology validators pass.

## 6. Suggested Playtest Axes

Each playtest run should ask about:

- Quest purpose continuity.
- Choice meaningfulness.
- Narrative flow.
- Variety/repetition.
- Resource pressure.
- Clue/omen readability.
- Ending and Quest Report clarity.
