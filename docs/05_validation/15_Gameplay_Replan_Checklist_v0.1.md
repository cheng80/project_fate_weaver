# Gameplay Replan Checklist v0.1

## 1. 목적 검증

- [ ] Run에 Quest 목적이 있는가?
- [ ] Quest가 주 목표, 보조 목표, 실패 조건을 가지는가?
- [ ] Quest가 관련 Region과 Storylet 가중치에 영향을 주는가?
- [ ] 실패해도 의미 있는 결과나 해금 가능성이 있는가?

---

## 2. Expedition Clock 검증

- [ ] day가 존재하는가?
- [ ] turn이 존재하는가?
- [ ] time_of_day가 존재하는가?
- [ ] act가 존재하는가?
- [ ] Day 종료 시 식량, 피로, 위험도 처리가 있는가?
- [ ] Quest 제한일이 반영되는가?

---

## 3. 3-Card Choice 검증

- [ ] 매 Turn 3장의 카드가 제시되는가?
- [ ] 카드가 서로 다른 역할을 가지는가?
- [ ] 카드가 Quest, Region, Storylet, Item, Clue, Economy, Status 중 일부와 연결되는가?
- [ ] 단순 버튼이 아니라 위험/보상 판단을 제공하는가?

---

## 4. Multi-Select 검증

- [ ] 1장 선택이 가능하다.
- [ ] 조건부 다중 선택이 가능하다.
- [ ] 카드 간 conflict가 처리된다.
- [ ] 카드 간 combo가 처리된다.
- [ ] 다중 선택 비용이 존재한다.

---

## 5. 온톨로지 검증

- [ ] Quest → Region 관계가 있다.
- [ ] Region → Storylet Pool 관계가 있다.
- [ ] Storylet → Card Candidate 관계가 있다.
- [ ] Card → Result 관계가 있다.
- [ ] Result → Resource/Status/Item/Clue/Next Event 관계가 있다.
- [ ] Item/Clue/Omen/Status가 카드 후보에 영향을 준다.

---

## 6. 경제/점수/평판 검증

- [ ] money가 거래, 정보 구매, 뇌물, 치료, 고용 중 하나 이상에 사용된다.
- [ ] reputation이 NPC/마을 반응에 영향을 준다.
- [ ] score가 Run Review에 반영된다.
- [ ] health/food가 장기 탐험과 실패 조건에 연결된다.

---

## 7. 콘텐츠 볼륨 검증

- [ ] Quest 최소 10개 목표
- [ ] Tutorial Quest 최소 2개 목표
- [ ] 전체 Storylet/Event 최소 60개 목표
- [ ] 전체 Card 후보 최소 150개 목표
- [ ] Item 최소 25개 목표
- [ ] Clue 최소 25개 목표
- [ ] Omen 최소 20개 목표
- [ ] Ending 최소 8개 목표

---

## 8. 테마 검증

- [ ] 저주가 메인 테마처럼 보이지 않는다.
- [ ] 저주는 상태/위험 요소 중 하나로만 표현된다.
- [ ] 중심은 Quest, 탐험, 선택, 자원, 경제, 평판, 후속 사건이다.
